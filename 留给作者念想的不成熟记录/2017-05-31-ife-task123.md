Task01

主要是一些标签的含义与使用，毕竟自己多少还是有些遗忘。  
### HTML5标签相关 ###
```
<header></header>         <!--页眉部分-->
<nav><nav>                <!--定义导航链接-->
<main></main>             <!--主要功能区。定义和用法：<main> 标签规定文档的主要内容。仅IE不支持；<main> 元素中的内容对于文档来说应当是唯一的。它不应包含在文档中重复出现的内容，比如侧栏、导航栏、版权信息、站点标志或搜索表单；注释：在一个文档中，不能出现一个以上的 <main> 元素。<main> 元素不能是以下元素的后代：<article>、<aside>、<footer>、<header> 或 <nav>。-->
<artical></artical>       <!--定义文章-->
<figure></figure>         <!--定义和用法:<figure> 标签规定独立的流内容（图像、图表、照片、代码等等）;figure 元素的内容应该与主内容相关，但如果被删除，则不应对文档流产生影响。-->
<figcaption></figcaption> <!--定义 figure 元素的标题。-->
<aside></aside>           <!--定义和用法: <aside> 标签定义其所处内容之外的内容。aside 的内容应该与附近的内容相关。-->
<footer></footer>         <!--定义和用法: <footer> 标签定义文档或节的页脚。<footer> 元素应当含有其包含元素的信息。页脚通常包含文档的作者、版权信息、使用条款链接、联系信息等等。您可以在一个文档中使用多个 <footer> 元素。-->
```
### 不要忘了这些知识点 ###
- 在`<header>`标签中的`<h1>`标题是网站标题，而在`<artical>`标签中的`<h1>`标题则是文章标题，后者哪怕没有CSS渲染，其大小也小于前者。  

- `<img>`标签要加上alt属性，对于屏幕阅读器而言有帮助。  

- `<table>`标签中的`<thead>;<tfoot>;<tbody>`写法顺序是头、尾、内容；`<caption>`标签作用是表格的标题。其中`<thead>;<tfoot>;<tbody>`中都是`<tr><th></th></tr>`或者`<tr><td></td></tr>`  

- 同组的单选按钮要加上一致的name属性，这样单选按钮才能生效。

Task02
------
### HTML相关知识点 ###
- ARIA（Accessible Rich Internet Applications，无障碍富因特网应用）；ARIA中规定了一个role属性，表示所在元素的用途；`<header class="clearfix" role="banner">`通过为其指定值为banner的role属性，可以告诉屏幕阅读器它的用途是保存横幅图片。

- ** `<form>`表单中每行最好嵌套进一个`<div>`中，这样方便设置样式！**

### CSS相关知识点 ###
- 设置有序列表项的样式`list-style: decimal;`//数字型的；

- 设置圆角矩形：`border-radius`定义和用法:border-radius 属性是一个简写属性，用于设置四个 border-\*-radius 属性。提示：该属性允许您为元素添加圆角边框！**例子**：`border-radius: 2em 1em 4em / 0.5em 3em;`；

- box-shadow 属性向框添加一个或多个阴影。语法： 
```
box-shadow: h-shadow(必须，水平阴影的位置，允许负值) v-shadow(必须，垂直阴影的位置，允许负值) blur(可选，模糊距离) spread(可选，阴影尺寸) color(可选，阴影颜色) inset(可选，将外部因应该为内部阴影);。
```
传送门：[CSS3阴影](http://www.w3school.com.cn/cssref/pr_box-shadow.asp)

- 清除浮动，解决因浮动而造成父元素高度塌陷。
HTML:
```
<div class="clearfix">
	<div class="fl">aaa</div>
	<div class="fl">bbb</div>
	<div class="fl"></div>
</div>
```
CSS:
```
.fl{
    width: 200px;
    height:400px;
    background: #000;
    margin-right: 20px;
    float: left;
}
/*清除浮动*/
.clearfix:before,
.clearfix:after {
    content: ".";
    display: block;
    height: 0;
    visibility: hidden;
}
.clearfix:after{
    clear: both;
}
.clearfix{
    zoom: 1;
} /* IE < 8 */
```

Task03
------
**实现三栏式布局，左右固定宽度，中间自适应。**  

#### 收获 ####
1. 为了避免浏览器被缩放的太小，导致内容溢出，可以对包含块设置`min-width`样式。
2. 自适应宽度神器：calc()函数+float，使用此方法，HTML可以不在乎顺序
3. 还有一种简洁的方式，传送门：[张鑫旭-三栏式布局-自身浮动法](http://www.zhangxinxu.com/wordpress/2009/11/%E6%88%91%E7%86%9F%E7%9F%A5%E7%9A%84%E4%B8%89%E7%A7%8D%E4%B8%89%E6%A0%8F%E7%BD%91%E9%A1%B5%E5%AE%BD%E5%BA%A6%E8%87%AA%E9%80%82%E5%BA%94%E5%B8%83%E5%B1%80%E6%96%B9%E6%B3%95/)

#### 有关绝对定位的坑 ####
绝对定位（元素在绝对定位和浮动之后，变成块级元素）导致的问题：  
- 包裹性，及元素使用absolute之后，父级元素的宽度变成内容宽度、

- 高度塌缩，元素绝对定位后，脱离文档流（上天啦，出现类似ps层级模式），导致父元素高度塌缩。这和浮动在表现上相似，但是解决法子只有固定高度。

#### 绝对定位定位点的知识 ####
- 当绝对定位元素只设置了，`position: absolute;`，而没有设置left和top等偏移值的时候。元素定位点是其在文档流中的位置，但是绝对定位元素依旧脱离了文档流，之后的绝对定位元素会忽略先前绝对定位元素，**一旦使用了绝对定位，思考之后的元素要当其不存在。**

- **当设置定位偏移值，如果包含块设置了padding值，则其偏移计算从padding开始**。

- 绝对定位的定位父元素是没有设置`position: static;`的父元素，如果父元素均没有定位，则为html根元素。
